HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The title element defines the tile of the webpage.
  • The base element fines the base URL for a webpage.
  • The link element connects an external resource to an HTML document.
  • The script element embeds client side scripts in an HTML file.
  • The charset element is the character set for the HTML document.
  • The h1 element is a heading tag.
  • The p element is a paragraph tag.
  • The ul element makes an unordered list.
  • the img element contains image info.
  • the br element creates a line of empty space.
  • the a element is an anchor that creates links to the current webpage or others.
  • the href tag points to the URL for the link.
  • the section element defines a section in an HTML document that contains thematic content.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Separation of Concerns refers to separating code into different files based on use.
  • A selector defines the element or attributes to which the rules, or declarations, apply.
  • declarations are rules that contain two components: The CSS property and its value.
  • DRY coding encourages developers to reuse or share pieces of code in order to reduce the number of lines of code written.
  • Class attributes allows us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • A period preceding any HTML element is designated as a universal class selector; the same element assigned to that class will gain the property assignments.
  • box shadow property adds shadow effects around the element's frame.

Git

  • git status checks what branch we are currently on
  • git checkout -b branch-name: creates new brnch and switches to it
  • git add -A moves changes from the working directory to the staging area.
  • git commit -m takes the staged snapshot and commits it the project history.
  • git clone creates a copy of an existing Git repository.
  • git checkout allows you to navigate existing branches.
  • git config sets configuration options for your Git installation.
  • git pull downloads a branch from a remote repository, then merges it into the current branch.
  • git push lets you move a local branch to another respository.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • API's provide functionality such as dynamically creating HTML and CSS styles, generating 3D graphics and audio samples, etc.
  • Variable types include a string, number, Boolean, array, and an object.
  • operators are mathematical symbols taht produces a result based on two values.
  • An assignment assigns a variable to a variable.
  • Conditionals are code structures used to test if an expression returns true or not.
  • An array is a group of related data.
  • for loops uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Functions are sets of intructions that tells the computer how to perform a certain task.
  • The two requirements of a function are that you must define it and then call it.
